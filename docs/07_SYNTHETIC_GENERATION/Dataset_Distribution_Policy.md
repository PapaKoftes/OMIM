## Purpose

Defines the target distributions for the synthetic dataset. These distributions ensure the dataset reflects the real diversity of panel manufacturing geometry, not a uniform random sample.

---

## Dataset Split

```yaml
dataset_split:
  total_samples: 1000       # v0.1.0 target
  train: 0.70               # 700 samples
  val:   0.15               # 150 samples
  test:  0.15               # 150 samples

split_policy:
  - Test set is frozen after first generation; never regenerated
  - Val set is frozen after first generation
  - Train set may be regenerated with different seed if needed
  - Each split has same invalid_ratio (0.30) to ensure balanced evaluation
```

## Validity Distribution

```yaml
validity_distribution:
  valid_samples:   0.70   # 70% pass all rules
  invalid_samples: 0.30   # 30% have at least one validation error

invalid_breakdown:
  MFG-001 edge_clearance:    0.25   # of invalid samples
  MFG-002 hole_too_close:    0.25
  MFG-003 drill_diameter:    0.15
  GEO-001 open_contour:      0.20
  MFG-004 pocket_too_narrow: 0.15
```

Rationale: Edge clearance and hole spacing are the most common real-world errors; over-represented in the distribution.

```yaml
  BLIND_HOLE:       0.08

  # Milled features (less common in basic panels)

  DADO:             0.02

  PROFILE_CUT:      1.0    # Every panel has exactly one (outer boundary)
```

## Panel Dimension Distribution

Panel dimensions are NOT sampled uniformly — they follow realistic industry distributions:

```yaml
panel_dimension_distribution:
  # Width: bimodal distribution centered on common cabinet widths
  width_mm:
    distribution: categorical
    values: [300, 400, 450, 500, 600, 800, 900, 1000, 1200]
    weights: [0.10, 0.10, 0.10, 0.15, 0.25, 0.10, 0.10, 0.05, 0.05]
    # 600mm width is most common (full base cabinet)
  
  # Height: cabinet-component distribution
  height_mm:
    distribution: categorical
    values: [300, 600, 720, 900, 1980, 2100, 2200]
    weights: [0.10, 0.20, 0.25, 0.10, 0.15, 0.10, 0.10]
    # 720mm (base cabinet body height) most common
  
  # Thickness: EN 309 standard thicknesses
  thickness_mm:
    distribution: categorical
    values: [12.0, 15.0, 16.0, 18.0, 22.0, 25.0]
    weights: [0.05, 0.10, 0.15, 0.55, 0.10, 0.05]
    # 18mm is dominant
```

## Feature Density Categories

Feature density is a first-class generation parameter, not an afterthought:

```yaml
feature_density_categories:
  sparse:
    range: 0–5 features per panel
    use_case: Simple back panels, shelves with no hardware
    weight_in_dataset: 0.20

  medium:
    range: 3–15 features per panel
    use_case: Standard side panels, base cabinet components
    weight_in_dataset: 0.60

    range: 10–30 features per panel
    use_case: Highly-featured side panels with full shelf pin rows
    weight_in_dataset: 0.20
```

## Feature Probability Weights Per Panel

These are **per-panel inclusion probabilities** (not dataset proportions). Each weight is the probability that a panel will contain at least one instance of that feature type. Different from `feature_type_distribution` which is dataset-level frequency.

```yaml
# Per-panel feature inclusion probabilities
# Derived from real panel manufacturing observations
feature_inclusion_weights:
  PROFILE_CUT: 1.00        # Every panel has an outer profile — not sampled, always present
  SHELF_PIN_HOLE: 0.60     # Very common in shelving and side panels
  CONFIRMAT_HOLE: 0.50     # Common joining holes
  THROUGH_HOLE: 0.45       # Generic holes, common
  DOWEL_HOLE: 0.40         # Common alignment holes
  HINGE_CUP_HOLE: 0.35     # Common in door and side panels
  RABBET: 0.20             # Common at panel edges
  GROOVE: 0.25             # Back panel grooves, less common
  POCKET: 0.15             # Less common in flat panels
  OPEN_SLOT: 0.10          # Specialized
  INTERNAL_CUTOUT: 0.05    # Rare (cutout windows, etc.)
```

## Complexity Targets

Each generated sample is assigned a complexity category based on feature count and type mix:

```yaml
complexity_targets:
  simple:
    weight: 0.25    # 25% of samples
    definition: "≤ 3 feature types, ≤ 5 total features, no milled features"
  
  medium:
    weight: 0.55    # 55% of samples
    definition: "2–5 feature types, 3–15 total features, may include grooves/pockets"
  
  complex:
    weight: 0.20    # 20% of samples
    definition: "> 5 feature types, or > 15 features, or both holes and milled features"
```

This distribution mirrors real shop workloads: most panels are medium complexity.

## Ambiguity Target

```yaml
ambiguity_target: 0.08   # 8% of samples contain at least one ambiguous feature

# Ambiguous feature: geometry that matches multiple feature classes
# within the confidence range [0.30, 0.75]
# Examples:
#   - 8mm circle that could be DOWEL_HOLE or THROUGH_HOLE
#   - Small closed pocket that could be a POCKET or THROUGH_POCKET
#   - 5mm circle isolated from any pattern (can't confirm shelf pin without row context)
```

Ambiguous features are intentionally included to benchmark how models handle uncertainty. They must be labeled `UNKNOWN_FEATURE` or with a flagged hypothesis list — never with artificial certainty.

## Anti-Patterns (Forbidden Distributions)

```python
# FORBIDDEN: Uniform distribution (unrealistic)
width = rng.uniform(50, 1200)  # Never — produces unrealistic dimensions

# FORBIDDEN: Over-representing rare features
invalid_ratio = 0.90  # Never — makes benchmark too easy

# FORBIDDEN: Single feature type dominance
# All samples are THROUGH_HOLE only — never; use full feature distribution

# REQUIRED: Realistic distribution
width = rng.choice([300, 400, 450, 500, 600, 800, 900, 1000, 1200],
                   p=[0.10, 0.10, 0.10, 0.15, 0.25, 0.10, 0.10, 0.05, 0.05])
```
