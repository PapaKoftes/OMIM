## Purpose

Defines the formal rules that constrain synthetic panel generation. The constraint grammar ensures that VALID synthetic samples are physically realizable and that INVALID samples contain only violations that real validation rules can detect.

---

## Grammar Overview

The constraint grammar operates at three levels:

1. **Panel-level constraints**: Panel dimensions, material, thickness
2. **Feature-level constraints**: Individual feature placement and sizing
3. **Inter-feature constraints**: Relationships between features

---

## Panel-Level Constraints

```
PANEL(width, height, thickness) is valid if:
  50mm ≤ width ≤ 1200mm
  50mm ≤ height ≤ 2400mm
  thickness ∈ {12, 15, 16, 18, 22, 25}mm
  width × height ≤ 3600 × 2100mm²  (nesting table limit)
```

---

## Feature Placement Constraints (Valid Samples)

### Circle Constraints

```
CIRCLE(cx, cy, r) is valid if:
  cx - r ≥ 8mm from each panel edge  (MFG-001: edge clearance)
  ∀ other circle j: dist(ci, cj) - ri - rj ≥ 3mm  (MFG-002: wall thickness)
  2r ∈ [3mm, 40mm]  (MFG-003: drill diameter range)
  cx - r ≥ 0mm and cx + r ≤ panel_width  (within panel)
  cy - r ≥ 0mm and cy + r ≤ panel_height  (within panel)
```

### Pocket/Contour Constraints

```
POCKET(polygon) is valid if:
  min_inscribed_diameter(polygon) ≥ 6mm × 1.2 = 7.2mm  (MFG-004)
  polygon is contained within panel boundary with 8mm clearance  (MFG-001)
  polygon.is_simple == True  (GEO-002)
  polygon.is_closed == True  (GEO-001)
  polygon.area ≥ 1mm²  (GEO-006)
```

## Feature-Specific Generation Rules

### SHELF_PIN_HOLE Generation

```python
# Must be generated in aligned groups on 32mm grid
def generate_shelf_pin_group(panel, rng):
    column_x = rng.choice([37.0, panel.width - 37.0])  # standard setback
    start_y = 37.0
    count = rng.integers(3, 10)
    
    return [
        FeatureSpec(
            feature_class="SHELF_PIN_HOLE",
            entity_type="CIRCLE",
            center=(column_x, start_y + i * 32.0),
            radius_mm=2.5,   # 5mm diameter
            layer="DRILL"
        )
        for i in range(count)
        if start_y + i * 32.0 + 2.5 <= panel.height - 8.0  # edge clearance
    ]
```

### HINGE_CUP_HOLE Generation

```python
# Fixed geometry: 35mm diameter, 22.5mm from edge
def generate_hinge_cup(panel, rng, edge: Literal["left", "right"]):
    x = 22.5 if edge == "left" else panel.width - 22.5
    y = rng.uniform(80.0, panel.height - 80.0)  # realistic placement
    return FeatureSpec(
        feature_class="HINGE_CUP_HOLE",
        entity_type="CIRCLE",
        center=(x, y),
        radius_mm=17.5,   # 35mm diameter
        layer="DRILL"
    )
```

## Invalidation Grammar (Invalid Samples)

Invalid samples are generated by applying a single violation to an otherwise valid panel:

```python
INVALIDATION_TYPES = {
    "edge_clearance":    apply_edge_clearance_violation,    # MFG-001
    "hole_too_close":    apply_hole_spacing_violation,      # MFG-002
    "hole_too_small":    apply_undersized_hole,             # MFG-003
    "open_contour":      apply_open_contour,                # GEO-001
    "pocket_too_narrow": apply_narrow_pocket,               # MFG-004
}

def apply_edge_clearance_violation(panel, features, rng):
    """Move a circle so its centroid is within 6mm of a panel edge."""
    target = rng.choice([f for f in features if f.entity_type == "CIRCLE"])
    edge_offset = rng.uniform(2.0, 7.9)  # less than 8mm clearance
    target.center = (edge_offset, target.center[1])
    target.is_valid = False
    target.violations = ["MFG-001"]
    return features
```

**Rule**: Each invalid sample has exactly one injected violation type. This ensures clean attribution in benchmark evaluation.

## Grammar Completeness

The grammar is designed so that:
1. Every valid sample passes all Layer 1 and Layer 2 rules
2. Every invalid sample fails exactly the rule corresponding to its injected violation
3. No valid sample accidentally fails any rule (generation constraints are strictly tighter than validation thresholds)

```python
# Safety margin: generation constraints use tighter bounds than validation rules
GENERATION_MARGIN = {
    "edge_clearance_mm": 8.0 + 0.5,    # 8.5mm > MFG-001 minimum 8.0mm
    "wall_thickness_mm": 3.0 + 0.5,    # 3.5mm > MFG-002 minimum 3.0mm
    "min_diameter_mm":   3.0 + 0.2,    # 3.2mm > MFG-003 minimum 3.0mm
}
```
